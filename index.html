Lets talk about let, const, var and scope

09/12/2020 by Christopher Pavelsek

Whether you are brand new to coding or have been coding for years, it is worth your time to read up on the declarations available in JavaScript and their best uses. A Quick refresher on the different declarations follows below:

"var"

“var” declares a variable containing information, similar to “let” and “const”, however “var” declarations can be of global scope when defined outside a function or function scope if used within a function block.

Global Scope Example:

        
          var car = “Volkswagen”;

          function carType(){
              var car = “Volkswagen”;
          }

Function Scope Example:

        
          function carType(){
            var car = “Volkswagen”;
            }

“var” can also be declared without being initialized. If you do not initialize the variable, it will then default to “undefined”. For instance below, the variable “shoe” will set itself to “undefined” since I didn’t set it to Nike or Adidas.

Ex: var shoe; //undefined

“var” variables can be re-declared and updated. 

        
          var car = “Volkswagen”; 
          car = “Toyota”;

or

        
          var car = “Volkswagen”; 
          var car = “Toyota”;

Having a solid understanding of how and why “var” is used to declare variables can help to better understand “let” or “const” and when to utilize those tools.

"let"

“let” declares variables similar to “var” however with “let” you can utilized Block Scope. This means that variables declared using “let” inside a block cannot be accessed from outside the block.

Block Scope Example:

        let car = “Toyota”; 
          function carType(){
          var car = “Volkswagen”;
          }

Console.log(car); // Toyota will populate since let was used to declare “Volkswagen” within our function.

“let” can be updated within its scope however it cannot be re-declared within its scope. “let” becomes a very useful choice if you are reusing a variable name and don’t want to run into scope issues.

"const"

“const” variables maintain their values no matter what. These variables can be used within block scope similarly to “let”. “const” variables cannot be updated or re-declared and as such every “const” declaration must be initialized at the time of declaration.

It's important to note that features are constantly being updated and expanded upon so pay attention to updates with JavaScript! 😊

Regular Expressions in JavaScript are objects that describe a pattern of characters. These expressions are very useful in searching for a text pattern or a text replacement operation. Below we will break down the syntax and investigate some useful examples of Regular Expressions!

Syntax: As shown below, regular expressions consist of a pattern enclosed between slashes.

/pattern/modifiers;

Example: 

        let re = /the/;

You can also call the constructor function of RegExp object as shown below.

Example:

We can use the “searsch()” method to search a string and return the index of the match of the Regular Expression we created about - let re = /the/;

One key to improving UI - Reducing Cognitive Load

9/02/20 by Christopher Pavelsek

Reducing Cognitive Load. To me, this is a very fancy way to say keep it simple; don’t overload your user with too much at one time. Make the users goal straightforward and obtainable while trying to make that process as easy as possible. If I were to go on a site to register for a ping pong tournament, I want that process to be as simple as possible with as little mental toll put on me as possible. If I’m struggling to get my goal accomplished or find it too difficult, I’m likely to scrap the whole thing and move on.

In order to reduce Cognitive Load on the end user as much as you can, try to keep the following ideas in mind.

Avoid visual overstimulation:

Having endless amounts of images, links, popups can be very distracting to the user. Its wonderful that we have so many design tools these days to engineer our wildest creations, however sometimes it can be over designed. A good thought to keep in mind is that Miller’s Law states that generally we can hold seven pieces of information in our short-term memory (https://xd.adobe.com/ideas/process/information-architecture/6-ways-to-reduce-cognitive-load-for-a-better-ui/). If you try to envision what that may look like when designing a website, it can help to give a cleaner appearance and better experience for the user.

Consistency:

A website that is designed uniformly across the board is the best way to organize a site. Fonts should match up from one place to another generally, icons shouldn’t change color from page to page, etc. One way to help with this is to offer less information or choices so that there is less chance for inconsistencies. One way to help with this would be a style guide where fonts and design decisions can be made in one location for simplicity sake. Below is an example of poor consistency.

The fonts are different from place to place, different sizes/colors and fonts all over. The alignment of the content also is all over the place which makes it harder to follow. This website will be less attractive to users or users who visit will be much less likely to stick around when they see the non-uniform layout.

The example above is a much better look at a consistent page design. The font from the logo on the top left corner matches the large center piece logo. The same blue/black tones match along the page and pop well against a lighter background. There isn’t a ton of information crowding the space either, it is straightforward and looks very efficient. I would say this is a much better idea of how to organize content.

Questionable Interface:

This one should be obvious to any designer, however its important to mention. A user shouldn’t have trouble trying to decide where they go to sign in or where they can click to find out more information. When you are creating a layout, things may appear obvious to you since you created them with your overall vision in mind; however it may not be apparent to most users. Try to put yourself in the mind of the average user or have someone test out the site just to understand their thought process across the page. The example below is a very minimalist design, which isn’t a bad take however the layout is a little strange and its not very clear where each button would take you or if they are all click able options.

All the examples we looked at today are aimed to reduce cognitive load for the end user. We want the experience to be as frictionless and thought free as possible for whoever is visiting the site. Keep these three key points in mind when you look to design your next page or even as you visit other pages across the web.

Pay Attention to Feedback

9/14/20 by Christopher Pavelsek

Feedback is such a huge part of the UI experience! When I sign into an account online, I like to see a positive affirmation that I have signed in. If I click to buy something, I expect to see some confirmation that my payment went through, and the order was processed. According to David Hogue, UX Design Lead at Google, feedback answers questions across four categories: location, current status, future status and outcomes/results. Below I will go more into detail on good examples of feedback in UI and how they contribute to positive design for the user.

Location:

This above example is a perfect example of locational feedback. There are many options available on this left hand navigation bar, however the feedback lets me know I am on the “diary” section. Also clearly listed would be the day, week, month or view selectors across the top bar. A nice bold red is used to distinguish the difference between the selected tab and non selected tabs. These two features allow for the user to know where they are at all times, making for a better UI experience.

Current Status:

A progress or status indicator is another great tool to keep the user engaged. The above picture is a great look at a loan tracking application/site. These processes can seem complicated or intimidating to a user however with a progression indicator like this, it's reassuring to the user to know exactly where they are at. As shown, you can see the indicator moves along from initial questions all the way to a completed deal.

Future Status:

Making sure that your UI updates the user on the future status of their task is very important. The example listed here is from H&M, a clothing retailer, and this is the feedback a user will see once they complete the order. The user is not left wondering what their status is after spending their hard earned money. This confirmation lets them know what will happen next and confirms the transaction. Without something like this it may be rather ambiguous to the user if the order completed or what the next steps may be.

Outcome/Results:

Inuit always presents a fantastic UI experience in my opinion. Turbo Tax for example displays constant feedback from start to finish for a tax return process. The example above is a final confirmation page recapping all the details of the tax return. It does a great job clearly listing at the top that the process has completed and below you can see the highlights from the process along with dates when they occurred. I always like the simplistic yet welcoming display that this website provides.

As you can see providing constant feedback to the user makes for a much more complete UI experience. If there is persistent feedback, you may tend not to notice it however you will certainly notice it if a website is lacking those updates.
